2水の波紋（純WebGL・外部依存なし）demo above

カーソル/タッチで波紋が広がり水面を屈折させる。WebGL拡張機能・外部ライブラリ不要で1ファイル完結。Mac Safari/Chrome 両対応。

  • A. 手続き生成版（背景画像なし）— lembrasser_01.html。ダークティールの水面＋睡蓮＋コースティクスをGLSL生成。
  • B. 実写テクスチャ版（写真を波打たせる）— lembrasser_26.html。写真を sampler2D に読み、波紋でUVを歪ます。画像は同一オリジンに置く（CORS対策）。

波紋の核 — 着水 → 因果的な波面 → 同心円が伝播

// uRipples[i] = vec4(中心x, 中心y, 開始時刻, 強さ)  ※xyはuv(0..1)
float rage = uTime - rp.z;
if (rage < 0.0 || rage > 6.43) continue;
float age = rage * 0.7;                 // ← 速度係数。0.7で「0.7倍速」スロー
float front  = age * 0.40;              // 因果的な波面（ここまで波が到達）
float behind = front - dist;            // >0 なら到達済み
float reach  = smoothstep(0.0, 0.04, behind);
float fb     = exp(-behind*behind*38.0);                 // 波面付近を強調
float wave   = sin(dist*46.0 - age*22.0)                 // 進行波
             + 0.4*sin(dist*92.0 - age*40.0);            // ＋毛細波（先端ほど細かい）
float amp    = exp(-dist*2.1) * exp(-age*1.1) * rp.w;    // 距離と時間で自然減衰
float w      = wave * amp * reach * (0.65 + 0.6*fb);
disp += dir * w;     // 変位（屈折）
hi   += max(w,0.0);  // 稜線ハイライト
act  += abs(w) + 0.22*amp*reach;                         // 波紋が広がった範囲
splash += exp(-dist*dist*950.0) * exp(-age*9.0) * rp.w;  // 着水点のきらめき

「波紋が広がった所をキラキラ明るい水色に」

vec3 sparkleAqua = vec3(0.32, 0.80, 0.88);
water = mix(water, sparkleAqua, clamp(act*0.55, 0.0, 0.52));     // 活性域を明色へ
water += vec3(0.45,0.90,0.98) * ca * act * 0.7;                  // コースティクス増幅
water += vec3(0.90,1.00,1.00) * pow(max(hi,0.0), 2.2) * 0.9;     // 稜線の白いきらめき
water += vec3(0.85,1.00,1.00) * splash * 1.1;                    // 着水点のきらめき

JS側の要点

  • フルスクリーン三角形 [-1,-1, 3,-1, -1,3] を1枚描く。
  • uRipples は長さ12のリングバッファ（未使用は z=-1）。
  • pointermove を ~28ms スロットリングし、速度から強さを決め spawn()。
  • Idle Ripples: 無操作でも1〜1.8秒ごとに水滴を落とす（水面が呼吸）。
  • 大画面対策: 内部解像度を min(1, 1280/innerWidth) で抑えCSSで拡大。
  • visibilitychange で停止、prefers-reduced-motion で静止。

実写版だけの追加（cover-fit）

float screenA = uRes.x/uRes.y, texA = uTexRes.x/uTexRes.y;
vec2 scale = (screenA > texA) ? vec2(1.0, texA/screenA) : vec2(screenA/texA, 1.0);
vec2 cuv = (uv + disp*0.04 - 0.5) * scale + 0.5;   // 画像を画面いっぱいにcover
vec3 col = texture2D(uTex, cuv).rgb;

テクスチャ読込時: UNPACK_FLIP_Y_WEBGL=true、CLAMP_TO_EDGE + LINEAR（NPOT対応）。フル実装は lembrasser_01.html / lembrasser_26.html。

3Liquid Glass（フロストガラス）

4万年筆で一文字ずつ書く演出

各文字を <span> に分割し、clip-path で左→右にインクを引き、blur でにじみ→定着。1文字ずつ遅延。

.ink-char{
  display:inline-block; opacity:0;
  clip-path:inset(0 100% 0 0);   /* 左→右に出現 */
  filter:blur(3px);              /* にじみ → 定着 */
  animation:penWrite .52s cubic-bezier(.5,.15,.25,1) forwards;
}
@keyframes penWrite{
  0%  { opacity:0; clip-path:inset(0 100% 0 0); filter:blur(3px); }
  55% { opacity:1; }
  100%{ opacity:1; clip-path:inset(0 0 0 0);   filter:blur(0); }
}
@media (prefers-reduced-motion: reduce){
  .ink-char{ animation:none; opacity:1; clip-path:none; filter:none; }
}

各 span に animation-delay を 0.2〜0.3秒刻みで。aria-label で読み上げ対応。手書きフォント（Klee One）と合わせると万年筆らしさUP。

5古い図鑑風の背景（Köhler 1887・PD）

100年以上前の薬用植物図譜 Köhler's Medizinal-Pflanzen (1887) はパブリックドメイン。アーユルヴェーダの薬草（ウコン/生姜/カルダモン/胡椒/シナモン/サフラン等）が実在し合法に使える。

# 1) ファイル名を検索（例: ウコン）
curl -s "https://commons.wikimedia.org/w/api.php?action=query&list=search&srsearch=Curcuma%20longa%20Medizinal-Pflanzen&srnamespace=6&format=json"

# 2) 安定した直URL（CDN・ホットリンク可）を取得
curl -s "https://commons.wikimedia.org/w/api.php?action=query&titles=File:<filename>&prop=imageinfo&iiprop=url&iiurlwidth=1000&format=json"
#    → thumburl が upload.wikimedia.org/.../thumb/... で返る
/* 古紙に溶け込ませる */
#botanical-bg img{
  mix-blend-mode:multiply;       /* 古紙の白を地に溶かし、図のインクだけ残す */
  filter:saturate(.66) contrast(.92) brightness(1.06) hue-rotate(-6deg);
  opacity:.42;
}

実証済みURL6種は lembrasser_25.html の #botanical-bg 内。暖色の図版を青へ寄せるには上に半透明グラデーション（cool wash）を重ねる。

6その他の小技

  • スクロール出現: .reveal{opacity:0; transform:translateY(26px); transition:…} ＋ IntersectionObserver で .in 付与。
  • グレイン: インラインSVG feTurbulence を mix-blend-mode multiply(紙)/screen(暗背景) で薄く。
  • 白文字を画像の上で読ませる: 暗いスクリム linear-gradient(rgba(...,.55), rgba(...,.62)) ＋ text-shadow。明るい写真＋白文字は必須。
  • 全テキスト2倍: html{ font-size:200% }（rem基準のTailwind text-* が一括2倍。※spacingも2倍になる）。
  • アクセシビリティ: 動く演出は必ず @media (prefers-reduced-motion: reduce) で静止/簡略化。

設計原則（効いたもの）

  1. 外部依存ゼロを優先（WebGLは生・画像はPDか自前・1ファイル完結）。
  2. AIは判断に、決定論はコードで（波紋の物理・スロットリング等）。
  3. 派手な演出ほど可読性とのトレードオフを明示し、スクリム等で担保。
  4. 破壊的変更の前にプレビュー→適用→検証（grep残存0など）。
