### 「全文字を手書きに」する黄金パターン
Caveat は欧文しか持たないので、**和文手書きフォントとペアにする**。フォントスタックの先頭に Caveat、次に Klee One を置くと、欧文＝Caveat・和文＝Klee One が自動で使い分けられる。

```css
/* 全テキストを手書きに（欧文Caveat＋和文Klee One） */
body, body * {
  font-family: 'Caveat', 'Klee One', cursive !important;
}
```

### Google Fonts 読み込み
```html
<link href="https://fonts.googleapis.com/css2?family=Caveat:wght@400;500;600;700&family=Klee+One:wght@400;600&family=Zen+Kurenaido&display=swap" rel="stylesheet" />
```

## 2. 水の波紋（純WebGL・外部依存なし）

カーソル/タッチで波紋が広がり、水面を屈折させる演出。**WebGL拡張機能・外部ライブラリ不要**で1ファイル完結。Mac Safari/Chrome 両対応。
2種類ある。

### A. 手続き生成版（背景画像なし）— `lembrasser_01.html`
ダークティールの水面＋睡蓮＋コースティクスをGLSLで生成し、その上に波紋を重ねる。

### B. 実写テクスチャ版（写真を波打たせる）— `lembrasser_26.html`
任意の写真を `sampler2D` に読み込み、波紋でUVを歪ませる。**画像は同一オリジンに置く**（CORSでテクスチャが汚染されないため）。

### 波紋の核（フラグメントシェーダーのループ）
リアルさのキモは「**着水 → 因果的な波面 → 同心円が伝播**」。波面より外は静止させる（causality）。

```glsl
// uRipples[i] = vec4(中心x, 中心y, 開始時刻, 強さ)   ※xyはuv(0..1)
float rage = uTime - rp.z;
if (rage < 0.0 || rage > 6.43) continue;
float age = rage * 0.7;                 // ← 速度係数。0.7で「0.7倍速」スロー
float front  = age * 0.40;              // 因果的な波面（ここまで波が到達）
float behind = front - dist;            // >0 なら到達済み
float reach  = smoothstep(0.0, 0.04, behind);
float fb     = exp(-behind*behind*38.0);                 // 波面付近を強調
float wave   = sin(dist*46.0 - age*22.0)                 // 進行波
             + 0.4*sin(dist*92.0 - age*40.0);            // ＋毛細波（先端ほど細かい）
float amp    = exp(-dist*2.1) * exp(-age*1.1) * rp.w;    // 距離と時間で自然減衰
float w      = wave * amp * reach * (0.65 + 0.6*fb);
disp += dir * w;     // 変位（屈折）
hi   += max(w,0.0);  // 稜線ハイライト
act  += abs(w) + 0.22*amp*reach;                         // 波紋が広がった範囲
splash += exp(-dist*dist*950.0) * exp(-age*9.0) * rp.w;  // 着水点のきらめき
```

### 「波紋が広がった所をキラキラ明るい水色に」する仕上げ
```glsl
vec3 sparkleAqua = vec3(0.32, 0.80, 0.88);
water = mix(water, sparkleAqua, clamp(act*0.55, 0.0, 0.52));     // 活性域を明色へ
water += vec3(0.45,0.90,0.98) * ca * act * 0.7;                  // コースティクス増幅
water += vec3(0.90,1.00,1.00) * pow(max(hi,0.0), 2.2) * 0.9;     // 稜線の白いきらめき
water += vec3(0.85,1.00,1.00) * splash * 1.1;                    // 着水点のきらめき
```

### JS側の要点
- フルスクリーン三角形 `[-1,-1, 3,-1, -1,3]` を1枚描く。
- `uRipples` は長さ12のリングバッファ（`Float32Array(12*4)`、未使用は z=-1）。
- `pointermove` を ~28ms スロットリングし、速度から強さを決めて `spawn()`。
- **Idle Ripples**: 無操作でも1〜1.8秒ごとに水滴を落とす（水面が呼吸する）。
- 大画面対策: 内部解像度を `min(1, 1280/innerWidth)` で抑え、CSSで引き伸ばす。
- `visibilitychange` で非表示時に停止。`prefers-reduced-motion` で静止。
- WebGL不可時は `canvas` を消すかCSS背景画像にフォールバック。

> フル実装（VERT/FRAG/ループ全部）は `lembrasser_01.html`（688〜910行付近）と `lembrasser_26.html`（実写テクスチャ版）を参照。

### 実写版だけの追加点（cover-fit）
```glsl
float screenA = uRes.x/uRes.y, texA = uTexRes.x/uTexRes.y;
vec2 scale = (screenA > texA) ? vec2(1.0, texA/screenA) : vec2(screenA/texA, 1.0);
vec2 cuv = (uv + disp*0.04 - 0.5) * scale + 0.5;   // 画像を画面いっぱいにcover
vec3 col = texture2D(uTex, cuv).rgb;
```
テクスチャ読込時は `gl.pixelStorei(gl.UNPACK_FLIP_Y_WEBGL, true)`、`CLAMP_TO_EDGE` + `LINEAR`（NPOT対応）。

## 3. Liquid Glass（フロストガラス／Apple風）

`backdrop-filter` のぼかし＋彩度、ふちのハイライト、液体的な光沢の擬似要素。`.lg`=透け強め（ナビ向き）、`.lg-card`=やや白め（本文の可読性確保）。

```css
.lg{
  position:relative; overflow:hidden;
  background:linear-gradient(135deg, rgba(255,255,255,.30), rgba(255,255,255,.10));
  backdrop-filter:blur(22px) saturate(180%);
  -webkit-backdrop-filter:blur(22px) saturate(180%);
  border:1px solid rgba(255,255,255,.55);
  box-shadow:0 10px 38px rgba(16,70,62,.20),
             inset 0 1px 0 rgba(255,255,255,.85),
             inset 0 -1px 0 rgba(255,255,255,.18);
}
.lg::after{                     /* 液体のような光沢 */
  content:""; position:absolute; inset:0; border-radius:inherit; pointer-events:none;
  background:linear-gradient(120deg, rgba(255,255,255,.55) 0%, rgba(255,255,255,0) 36%,
            rgba(255,255,255,0) 64%, rgba(255,255,255,.30) 100%);
  mix-blend-mode:screen;
}
/* 本文カード（読みやすさ優先・ガラス感は維持） */
.lg-card{
  background:linear-gradient(135deg, rgba(255,255,255,.52), rgba(247,253,251,.34));
  backdrop-filter:blur(18px) saturate(165%);
  border:1px solid rgba(255,255,255,.6);
  box-shadow:0 12px 40px rgba(16,70,62,.16), inset 0 1px 0 rgba(255,255,255,.8);
}
```
透明度・ぼくし量（`rgba`の0.30/0.10 と `blur()`）で印象が変わる。背景が動く（波紋など）と「液体ガラス」感が出る。

## 4. 万年筆で一文字ずつ書く演出 — `lembrasser_01.html`

各文字を `<span>` に分割し、`clip-path` で左→右にインクを引き、`blur` でにじみ→定着。1文字ずつ遅延。

```css
.ink-char{
  display:inline-block; opacity:0;
  clip-path:inset(0 100% 0 0);   /* 左→右に出現 */
  filter:blur(3px);              /* にじみ → 定着 */
  animation:penWrite .52s cubic-bezier(.5,.15,.25,1) forwards;
}
@keyframes penWrite{
  0%  { opacity:0; clip-path:inset(0 100% 0 0); filter:blur(3px); }
  55% { opacity:1; }
  100%{ opacity:1; clip-path:inset(0 0 0 0);   filter:blur(0); }
}
@media (prefers-reduced-motion: reduce){
  .ink-char{ animation:none; opacity:1; clip-path:none; filter:none; }
}
```
```html
<h1 class="handwrite" aria-label="本来の私へ"
><span class="ink-char" style="animation-delay:.55s">本</span
><span class="ink-char" style="animation-delay:.83s">来</span
><span class="ink-char" style="animation-delay:1.11s">の</span
><span class="ink-char" style="animation-delay:1.39s">私</span
><span class="ink-char" style="animation-delay:1.67s">へ</span></h1>
```
※ `aria-label` で読み上げ対応。手書きフォント（Klee One）と合わせると万年筆らしさが増す。

## 5. 古い図鑑風の背景（Köhler 1887・パブリックドメイン） — `lembrasser_25.html`

100年以上前の薬用植物図譜 **Köhler's Medizinal-Pflanzen (1887)** はパブリックドメイン。アーユルヴェーダの薬草（ウコン/生姜/カルダモン/胡椒/シナモン/サフラン等）が実在し、合法に使える。

### Wikimedia Commons から確実に取得する手順
```bash
# 1) ファイル名を検索（例: ウコン）
curl -s "https://commons.wikimedia.org/w/api.php?action=query&list=search&srsearch=Curcuma%20longa%20Medizinal-Pflanzen&srnamespace=6&format=json"

# 2) 安定した直URL（CDN）を取得
curl -s "https://commons.wikimedia.org/w/api.php?action=query&titles=File:<filename>&prop=imageinfo&iiprop=url&iiurlwidth=1000&format=json"
#    → thumburl が upload.wikimedia.org/.../thumb/... で返る（ホットリンク可）

# 簡易版: Special:FilePath（リダイレクト・レート制限あり。本番はupload直URL推奨）
# https://commons.wikimedia.org/wiki/Special:FilePath/<filename>?width=900
```
実証済みURL（6種）は `lembrasser_25.html` の `#botanical-bg` 内に記載。

### 古紙に溶け込ませる表示
```css
#botanical-bg img{
  mix-blend-mode:multiply;       /* 古紙の白を地に溶かし、図のインクだけ残す */
  filter:saturate(.66) contrast(.92) brightness(1.06) hue-rotate(-6deg);
  opacity:.42;
}
```
暖色の図版を**青へ寄せて統一**したい時は、上に半透明グラデーション（cool wash）を重ねる。

## 6. その他の小技

- **スクロール出現**: `.reveal{opacity:0; transform:translateY(26px); transition:…}` ＋ `IntersectionObserver` で `.in` 付与。
- **グレイン（紙/宇宙の塵）**: インラインSVGの `feTurbulence` を `mix-blend-mode:multiply`(紙) / `screen`(暗背景) で薄く敷く。
- **白文字を画像の上で読ませる**: 背景に暗いスクリム（`linear-gradient(rgba(...,.55), rgba(...,.62))`）＋ `text-shadow` を重ねる。明るい写真＋白文字は必須。
- **全テキスト2倍**: `html{ font-size:200% }`（rem基準のTailwind text-*が一括で2倍。※spacingも2倍になる点に注意）。
- **アクセシビリティ**: 動きのある演出は必ず `@media (prefers-reduced-motion: reduce)` で静止/簡略化。

## 設計原則（このプロジェクトで効いたもの）
1. **外部依存ゼロ**を優先（WebGLは生で書く・画像はPDか自前・1ファイル完結）。
2. **AIは判断に、決定論はコードで**（波紋の物理・スロットリング等はコード）。
3. 派手な演出ほど**可読性とのトレードオフを明示**し、スクリム等で担保する。
4. 破壊的変更の前にプレビュー→適用→検証（grep残存0など）。
